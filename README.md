# Swastik Bhardwaj - Portfolio

Hey! Welcome to the repository for my personal portfolio website.

This is a highly interactive, animated portfolio built to showcase my projects, skills, and experience, while also serving as a playground for some fun frontend experiments (and a few hidden easter eggs). 

## 🛠️ Tech Stack

- **Frontend:** React + TypeScript
- **Build Tool:** Vite
- **Styling:** Tailwind CSS
- **Animations:** Framer Motion
- **UI Components:** shadcn/ui & Radix UI
- **Icons:** Lucide React

## ✨ What's Inside?

Instead of just a static resume, I wanted this site to feel alive. Some core features include:
- **Hero & Galaxy:** Interactive backgrounds that react to your cursor.
- **Projects Showcase:** Clean UI for listing my full-stack and frontend work.
- **Testimonials & Experience:** Timeline and carousel of my journey.
- **Dev Console:** A fully working mini-terminal built into the site.

### 🥚 Easter Eggs (Spoilers!)

I hid a few fun things around the site just for the dev folks who like to click around:

1. **The Terminal Console:** 
   Hit the terminal icon (or press `~`) to open the dev console. Try running commands like `help`, `projects`, `about`, or if you're feeling dangerous... `sudo rm -rf /`.
2. **The "Fatal Error" Crash:** 
   In the Projects section, find the card for this "Portfolio Website" and hit the **Live Demo** button. Instead of opening a link (because you're already here), it triggers a fake system crash with shaking windows and terminal error logs. Just click anywhere on the screen to restore the system.
3. **Profile Pic Spinner:**
   Try clicking my profile photo exactly 4 times in a row.

## 🚀 Running it Locally

If you want to clone this and check out the code, it's pretty straightforward:

1. Clone the repo:
   ```bash
   git clone https://github.com/swastik7781/portfolio-professional.git
   ```
2. Install the dependencies:
   ```bash
   npm install
   ```
3. Spin up the dev server:
   ```bash
   npm run dev
   ```
4. Open your browser and go to the local port Vite gives you (usually `http://localhost:5173`).

## 🌐 Deployment

The site is configured to deploy automatically via **Vercel** straight from the `main` branch. Push to `main`, and it goes live.
